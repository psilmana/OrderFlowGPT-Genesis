# Release Notes — 0.1.1

## Milestone 6: Vision Object Detection Foundation

This release adds the object-detection architecture that later milestones will use to identify trading-screen semantics such as price text, bid/ask values, footprint cells, volume-profile structures, big trades, absorption, and stacked imbalances.

Milestone 6 is intentionally contract-only. It does not perform real object detection and does not introduce OpenCV, OCR, machine learning, AI calls, screen capture, external libraries, threading, async execution, networking, persistence, or global runtime state.

## Highlights

- Immutable object-detection dataclasses:
  - `ObjectId`
  - `ObjectType`
  - `DetectionConfidence`
  - `DetectionSource`
  - `DetectedObject`
- `DetectionGraph` for all detected objects in one frame, including validation for:
  - unique ids
  - duplicate ids
  - valid parent references
  - valid child references
  - frame-id alignment
- `DetectionContext` containing only:
  - `ProcessedFrame`
  - `WorkspaceLayout`
  - immutable configuration mapping
- Object-detection interfaces and orchestration contracts:
  - `ObjectDetector`
  - `ObjectDetectionPipeline`
  - `DetectorRegistry`
  - `SequentialObjectDetectionPipeline`
- Placeholder detectors that return empty `DetectionResult[DetectedObject]` values:
  - `PriceAxisDetector`
  - `TimeAxisDetector`
  - `FootprintDetector`
  - `VolumeProfileDetector`
  - `BigTradeDetector`
  - `AbsorptionDetector`

## Roadmap Fit

Milestone 6 sits between chart/workspace understanding and real semantic object detection. It preserves the Milestone 4 `WorkspaceLayout` contract and the Milestone 5 chart detector flow while adding a validated graph model for future Milestone 7 detectors.
