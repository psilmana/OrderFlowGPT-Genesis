# Release Notes - 0.1.9

## Milestone 14 - OCR Post Processing & Numeric Interpretation

Milestone 14 converts Milestone 13 raw OCR text into validated numeric values. Genesis now provides immutable `ParsedValue`, `NumericValue`, `NumericType`, `ParsingResult`, `ParsingError`, `NormalizationRule`, and `OCRNormalizationConfiguration` models, plus `OCRPostProcessor`, `NumericParser`, and `NormalizationPipeline` contracts.

The deterministic post-processing path is raw `OCRResult` text, normalization, validation, numeric parsing, and `ParsedValue`. Normalization supports configured character replacement, OCR confusable replacement such as O/o to 0 and I/l to 1, whitespace removal, thousands-separator removal, Unicode minus normalization, decimal separator normalization, duplicate-sign cleanup, and leading/trailing garbage trimming.

`DeterministicNumericParser` accepts integers, decimals, signed integers, and signed decimals. It rejects malformed values, invalid decimals, alphabetic values after normalization, duplicate signs, multiple decimal points, empty normalized text, NaN, Infinity, and configured length overflow.

`DetectionGraph` now exposes `parsed_values`, `lookup_parsed(cell_id)`, `lookup_numeric(cell_id)`, and `lookup_invalid()` while keeping existing raw OCR helpers intact.

Milestone 14 still has no trading concepts. Genesis does not know bid, ask, delta, volume, POC, imbalance, absorption, or any market semantics. Milestone 15 introduces Footprint Semantic Interpretation.
